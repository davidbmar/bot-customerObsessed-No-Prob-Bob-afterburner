`★ Insight ─────────────────────────────────────`
The `updateSendButton()` function implements a state machine for the Send button slot with clear priority:
1. **Streaming** (highest) — always shows ⏹ Stop, regardless of hands-free state
2. **Hands-free + no text** — shows ⏸/▶ Pause/Resume
3. **Normal** — shows Send (enabled/disabled based on input)

This single function is called from every state transition point (stream start/end, handsfree toggle, input change, abort), making the button behavior deterministic and avoiding race conditions between states.
