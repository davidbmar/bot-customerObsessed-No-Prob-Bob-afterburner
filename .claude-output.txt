`★ Insight ─────────────────────────────────────`
- **Code block extraction before splitting**: The original `renderMarkdownToDOM` split all text by `\n\n+` first, then tried to match code blocks. This broke any fenced code block containing blank lines (common in real code). The fix extracts code blocks in a first pass, then splits the remaining text segments.
- **XSS-safe rendering**: The entire markdown renderer uses `document.createElement()` + `textContent` exclusively — never `innerHTML`. This means even if an LLM outputs `<script>alert('xss')</script>`, it renders as literal text, not executable HTML.
- **Date grouping state management**: Every place that clears the message container (`newConversation`, `clearHistory`, `switchToConversation`, `deleteConversation`, `restoreConversation`) must also reset `lastMessageDateStr`, otherwise date separators won't appear after switching contexts.
