`★ Insight ─────────────────────────────────────`
**Regex backreference gotcha**: The original repeated-word regex `^(\w+\s*)\1{2,}$` captures the trailing space as part of the group, so the backreference `\1` requires that exact space count in every repetition — including the last one before `$`. The fix `^(\w+)(?:\s+\1){2,}\s*$` separates the word capture from the whitespace, so "the the the" matches because `\1` is just "the" and `\s+` handles inter-word spacing independently.

**Filtered responses as 200, not errors**: Returning `{"filtered": true}` with HTTP 200 (not 4xx) is intentional — the client recorded audio, the server transcribed it successfully, the result just wasn't worth sending to the LLM. This lets the UI silently drop garbage without showing error states to the user.
