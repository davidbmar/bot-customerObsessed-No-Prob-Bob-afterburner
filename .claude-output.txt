`★ Insight ─────────────────────────────────────`
- **data-attribute pattern**: The codebase uses `dataset.markdown` as a shadow copy of the original markdown source text. This is a common pattern in chat UIs — the rendered HTML goes in `.innerHTML` for display, but the raw source is stashed in a `data-*` attribute so it can be re-rendered later (e.g., after restore from localStorage). The bug was that `saveConversation()` read `.textContent` (which strips formatting) instead of the stashed source.
- **Event-driven sidebar refresh**: Rather than using MutationObservers or a pub/sub system, the codebase uses explicit `refreshConversationList()` calls at mutation points. This is simpler but requires remembering to add the call at every place that changes state — exactly the kind of thing that gets missed (as it was here in `saveConversation` and `newConversation`).
