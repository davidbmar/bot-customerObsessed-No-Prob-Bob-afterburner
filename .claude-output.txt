`★ Insight ─────────────────────────────────────`
**Key design pattern used:** Splitting `stopAgentSpeaking()` into two functions (`stopAgentSpeakingOnly` + `stopAgentSpeaking`) separates "stop TTS audio" from "stop TTS + pause mic". This is important because the same "stop speaking" action has different intent depending on context:
- **User-initiated stops** (Escape, pause button, stop banner): user wants silence — pause everything
- **Barge-in** (onSpeechStart, push-to-talk): user is actively speaking — stop bot but keep listening

Without this split, barge-in would pause the mic right as the user tries to talk, defeating its purpose.
