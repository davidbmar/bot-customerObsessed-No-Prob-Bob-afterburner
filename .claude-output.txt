`★ Insight ─────────────────────────────────────`
**State machine pattern for UI transitions**: The voice preview flow uses a simple state machine (`showVoicePreview` → timer → `confirmVoicePreview` OR user action → `discardVoicePreview`). The key design choice is that `onVoicePreviewEdit` cancels the auto-send timer but keeps the discard UI visible — this lets users edit without time pressure while still being able to discard.

**Garbled detection heuristic**: Rather than a complex NLP check, a simple ratio of alphabetic characters to total length (<40% alpha = garbled) catches most noise transcriptions (random punctuation, numbers from TV captions) while letting real speech through.
