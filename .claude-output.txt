`★ Insight ─────────────────────────────────────`
The streaming typing dots implementation uses a key pattern: **optimistic UI with graceful cleanup**. The dots are added to the DOM immediately when the streaming bubble is created, then removed via three separate paths: (1) on first token arrival (normal flow), (2) when stream ends without a done event (fallback), and (3) implicitly when the done event clears and re-renders the body with markdown. This defensive approach ensures dots never get "stuck" regardless of how the stream terminates.
