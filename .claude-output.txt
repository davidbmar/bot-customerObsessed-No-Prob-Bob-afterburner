`★ Insight ─────────────────────────────────────`
**Integration test pattern**: The tests use `create_app(gateway, port=0)` which lets the OS pick a free port — this avoids port conflicts in CI. The `live_server` fixture starts a real threaded HTTP server so we test actual HTTP behavior (headers, status codes, content) rather than mocking at the handler level. This gives higher confidence than unit tests for routing changes.
