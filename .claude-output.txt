`★ Insight ─────────────────────────────────────`
**VAD restart strategy:** Silero VAD (via `@ricky0123/vad-web`) doesn't expose a method to update options on a running instance. The approach here — `stopVAD()` then `setTimeout(startVAD, 200)` — gives the old instance time to release the microphone before the new one requests it. Without this delay, `getUserMedia` could fail with a "device busy" error on some browsers.

**Barge-in guard placement:** The guard was added in *two* places — the VAD `onSpeechStart` callback (hands-free mode) and the `startVoiceRecording` function (push-to-talk mode). This ensures consistent behavior regardless of input method.
