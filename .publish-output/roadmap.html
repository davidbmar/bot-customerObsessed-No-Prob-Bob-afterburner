Key Architectural Decisions

  1. Personality as markdown, not config. Humans write documents, LLMs read them. No YAML schemas, no rigid templates. Validated across 11 sprints and 22 features.
  1. Principles over patterns. Encode the spirit ("understand the problem before proposing solutions"), not exact phrases. The LLM has judgment вҖ” let it use it.
  1. Gateway pattern for LLM abstraction. Multiple transports feed into one gateway, which routes to one of N providers. Validated when Claude was added in Sprint 11 with zero transport changes.
  1. Memory as a separate system. Not embedded in personality. Reusable across all bots. JSONL for history, JSON for extracted facts.
  1. Personality inheritance. base.md defines foundation behavior. Domain personalities extend it. New bots inherit base automatically.
  1. Scenario-based evaluation. Test bot behavior with conversation scenarios, not unit tests. "Given this input, does the bot follow its principles?"
  1. Single-agent sprints. After merge failures in Sprints 4-5, switched to single-agent sprints for reliability. This pattern held through Sprint 11.
