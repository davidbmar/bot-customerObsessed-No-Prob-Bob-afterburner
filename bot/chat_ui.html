Architecture

┌──────────────────────────────────────────┐
│       python3 -m bot.server (:1203)      │
│                                          │
│  Web Chat ─┐                             │
│  Telegram ─┼──▶ Gateway ──▶ LLM         │
│  CLI ──────┘    │             │          │
│                 │    Ollama / Claude /    │
│            Memory    ChatGPT             │
│            Tools (6 Afterburner APIs)    │
│                                          │
│  Voice Pipeline        Auth + Storage    │
│  ├─ STT: Whisper       ├─ Google SSO    │
│  ├─ TTS: Piper         ├─ SQLite        │
│  ├─ Input Filter       └─ Sessions      │
│  └─ VAD (browser)                        │
└──────────────────────────────────────────┘

All interfaces share the same brain. Switching LLM in the web UI changes it for Telegram too.

Quick Start

  1. Install Ollama and run ollama pull qwen3:4b
  2. Clone: git clone https://github.com/davidbmar/bot-customerObsessed-No-Prob-Bob-afterburner.git
  3. Install: python3 -m venv .venv && source .venv/bin/activate && pip install -e ".[dev]"
  4. Start: bash scripts/start.sh
  5. Open: http://localhost:1203/chat

Voice

Push-to-talk: Hold the mic button to speak. Release to send.

Hands-free: Toggle on — speaks naturally, auto-detects speech via VAD.

Echo cancel: Mic pauses while bot speaks back.

Input filter: Garbage ("um", silence, noise) is dropped before hitting the LLM.

Telegram

  1. Message @BotFather on Telegram, send /newbot
  2. Add token to .env: TELEGRAM_BOT_TOKEN=your-token
  3. Restart server — Telegram polling starts automatically

Key API Endpoints

POST /api/chat/streamChat with SSE streaming
POST /api/voice/transcribeSpeech to text
POST /api/voice/synthesizeText to speech
POST /api/llm/switchSwitch LLM provider
GET /api/healthServer status
POST /api/tools/save_discoverySave as seed doc
POST /api/auth/googleGoogle Sign-In
