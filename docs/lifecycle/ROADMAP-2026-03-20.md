## Key Architectural Decisions

1. **Personality as markdown, not config.** Humans write documents, LLMs read them. No YAML schemas, no rigid templates. Validated across 11 sprints and 22 features.

2. **Principles over patterns.** Encode the spirit ("understand the problem before proposing solutions"), not exact phrases. The LLM has judgment — let it use it.

3. **Gateway pattern for LLM abstraction.** Multiple transports feed into one gateway, which routes to one of N providers. Validated when Claude was added in Sprint 11 with zero transport changes.

4. **Memory as a separate system.** Not embedded in personality. Reusable across all bots. JSONL for history, JSON for extracted facts.

5. **Personality inheritance.** base.md defines foundation behavior. Domain personalities extend it. New bots inherit base automatically.

6. **Scenario-based evaluation.** Test bot behavior with conversation scenarios, not unit tests. "Given this input, does the bot follow its principles?"

7. **Single-agent sprints.** After merge failures in Sprints 4-5, switched to single-agent sprints for reliability. This pattern held through Sprint 11.
